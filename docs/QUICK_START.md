# 快速开始指南

## 5 分钟上手 VESA 时序计算器

### 步骤 1: 启动应用程序

```bash
python vesa_timing_calculator.py
```

### 步骤 2: 选择计算模式

在"计算模式"下拉菜单中选择：
- **从刷新率计算像素时钟** - 设计新时序
- **从像素时钟计算刷新率** - 分析现有时序

### 步骤 3: 输入参数

#### 方式 A: 使用预设（推荐新手）
1. 在"预设"下拉菜单选择分辨率
2. 自动填充参数并计算

#### 方式 B: 手动输入
1. 输入水平分辨率（如 1920）
2. 输入垂直分辨率（如 1080）
3. 输入刷新率（如 60）或像素时钟（如 148.5）
4. 可选：勾选 "Reduced Blanking"

### 步骤 4: 查看结果

计算结果会实时显示在右侧表格中，包括：
- 像素时钟
- 水平时序参数（总像素、消隐区、前廊、同步、后廊）
- 垂直时序参数（总行数、消隐区、前廊、同步、后廊）

### 步骤 5: 使用结果

#### 复制结果
点击"复制所有结果"按钮，将结果复制到剪贴板

#### 生成 RTL 代码
1. 点击"生成 RTL 代码"按钮（绿色）
2. 查看成功提示
3. 在 `./output` 目录找到生成的文件

## 常见使用场景

### 场景 1: 设计 1080p60 显示时序

```
1. 选择模式: "从刷新率计算像素时钟"
2. 选择预设: "1920x1080@60Hz"
3. 查看结果: 像素时钟 ≈ 147.84 MHz
4. 点击"生成 RTL 代码"
```

### 场景 2: 分析 HDMI 时序

```
1. 选择模式: "从像素时钟计算刷新率"
2. 输入分辨率: 1920x1080
3. 输入像素时钟: 148.5 MHz
4. 查看结果: 刷新率 ≈ 60.21 Hz
```

### 场景 3: 生成 4K 时序 RTL

```
1. 选择模式: "从刷新率计算像素时钟"
2. 输入分辨率: 3840x2160
3. 输入刷新率: 60 Hz
4. 勾选 "Reduced Blanking"
5. 点击"生成 RTL 代码"
```

## 生成的文件说明

### RTL 模块文件
`output/vesa_timing_<分辨率>_<刷新率>hz.v`

包含完整的时序生成器模块，可直接用于 FPGA 项目。

### 测试平台文件
`output/tb_vesa_timing_<分辨率>_<刷新率>hz.v`

包含测试平台代码，用于仿真验证。

## 仿真验证（可选）

### 使用 Icarus Verilog

```bash
# 进入输出目录
cd output

# 编译
iverilog -o sim vesa_timing_1920x1080_60hz.v tb_vesa_timing_1920x1080_60hz.v

# 运行仿真
vvp sim

# 查看波形
gtkwave tb_vesa_timing_1920x1080_60hz.vcd
```

## 批量生成（高级）

使用演示脚本批量生成多个分辨率：


这将生成 6 个常见分辨率的 RTL 代码。

## 常见问题

**Q: 如何修改生成的代码？**
A: 生成的代码是标准 Verilog，可以用任何文本编辑器修改。

**Q: 支持哪些 FPGA？**
A: 支持所有主流 FPGA（Xilinx、Intel/Altera、Lattice 等）。

**Q: 如何验证时序正确性？**
A: 使用提供的测试平台进行仿真，检查同步信号时序。

**Q: 可以生成其他语言的代码吗？**
A: 当前只支持 Verilog，未来可能支持 VHDL。
